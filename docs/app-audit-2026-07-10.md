# Application audit — 2026-07-10

## Scope

This audit combines a read-only code review with hands-on Playwright CLI exploration of the live dashboard at desktop and mobile sizes. It covers the web dashboard, project grouping, manual merge import, setup UI, report payloads, tests, documentation, and adjacent product direction.

The following were exercised in Chrome through Playwright CLI:

- Overview, Sessions, Models, Projects, and Sync routes;
- light and dark themes;
- URL-backed search and date filters;
- session drawer navigation and empty states;
- a generic mobile viewport (361 × 735 CSS pixels).

No real pairing, merge, import, group write, or other data-mutating UI operation was triggered during the audit.

## Adjacent independently specified work

The branch containing these audit follow-ups also delivers the Skills management control plane. That product surface is not an audit finding or part of the audit batch described below; its scope, safety requirements, ownership, and verification contract are defined separately in [`docs/skills-management-spec.md`](skills-management-spec.md).

## Baseline observations

With 3,633 sessions loaded, the initial route response was 15.55 MB and the first live refresh response was 18.14 MB. The Overview rendered 4,895 DOM nodes, including 396 calendar-day buttons and 1,746 scatter points. It exposed 432 tab stops.

The visual system is coherent, both themes are polished, exploration state persists well in the URL, the `/` shortcut is useful, and the session drawer provides good detail. The main weaknesses are payload growth, metric trust, mobile information architecture, chart accessibility, and several missing data/security invariants.

## Implementation outcome

- The web DTO now transports session rows once, validates the remaining facet payload without cloning the report, preserves limited standalone exports, and waits for the normal refresh interval when loader data is already current. On the deterministic fixture, removing the duplicate row array reduces serialized payload size by 21.9%.
- The Overview hero now separates estimated API-equivalent value, reported actual spend, and coverage. It makes no savings, ROI, or leverage claim.
- Cursor import artifacts have their own provenance field; orphan exports stay unattributed. Project-group IDs and source ownership are globally unique, and the editor moves sources rather than duplicating them.
- Merge bundles now have a strict schema, semantic numeric constraints, verified hashes/identity, and machine binding at both parsing and storage boundaries. Manual imports are loopback-only, same-origin JSON, bounded to 64 MiB and 50,000 rows, and protected from DNS rebinding.
- The web LAN pairing and peer-merge surface was removed. `/sync` retains bounded, explicit merge bundle import/export by file.
- Sessions and Projects have mobile summaries, mobile sorting, paginated keyboard access, and touch-sized controls. Timeline controls reflow at narrow widths. The heatmap uses roving focus with axis-correct arrow navigation; dense scatter data is aggregated per harness and exposes a structured accessible summary.
- Campaign projections and sorting are reused, timeline bars are built only for visible sparse entries, and the redundant TanStack sorting pass is removed.
- Pull-request CI now runs unit/integration tests, the production build, and five deterministic Playwright flows, including the 361-pixel mobile experience and the file-only Sync route.
- Setup rendering uses DOM text sinks, textual CSV cells are formula-neutralized, impossible/reversed dates are rejected, and contributor documentation matches current scripts.

## Product direction

These are options, not committed implementation scope for this batch.

### Metric-level trust

Extend the existing row-level provenance model to aggregate cards and deltas. Each headline metric should be able to explain how many rows were priced, partial, ambiguous, usage-unavailable, or missing an actual-cost signal. Avoid a single global “data quality” score.

### Harness health onboarding

Expose harness status as a compact health view: installed/not found, last successful collection, partial-data limitations, and the next useful action. With LAN pairing retired, keep Sync focused on local harness health and bounded manual import/export, with diagnostics behind progressive disclosure.

### Assisted project grouping

Bring the existing setup heuristics into the main Projects UI: suggest sources sharing a Git remote or stable basename, de-prioritize ephemeral agent worktrees, and exclude import artifacts. Suggestions should remain explicit and reversible.

### Shareable recap, later

A filter-aware “Wrapped” PNG/PDF could provide a strong product payoff, but it should follow the metric-trust work. It must default to redacted project/session identity and avoid ROI or break-even claims.
